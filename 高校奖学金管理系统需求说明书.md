# 高校奖学金管理系统需求说明书

## 1. 项目概述

### 1.1 项目背景
为规范高校奖学金评定流程，提高管理效率，保障奖学金评定的公平、公正、公开，开发一套功能完善、安全可靠的高校奖学金管理系统。

### 1.2 项目目标
- 实现学生信息的全面管理和学业成绩跟踪
- 构建完整的奖学金申请、评审、公示流程
- 提供精细化的权限管理和安全控制
- 支持数据备份、恢复和软删除功能
- 实现系统数据的初始化和自动导入

### 1.3 术语定义
- **奖学金类型**：国家奖学金、校级奖学金等不同级别的奖励类型
- **评审流程**：申请、初审、复审、公示、发放的完整流程
- **学业成绩**：课程成绩、学分、GPA等学习表现指标
- **软删除**：不真正删除数据，而是标记数据为已删除状态
- **权限管理**：基于角色的访问控制，限制用户对系统资源的访问

## 2. 技术架构

### 2.1 技术栈
- **后端**：Spring Boot 3.1.5 + Spring MVC + Spring Data JPA
- **前端**：Thymeleaf 3.1 + Bootstrap 5.3 + jQuery 3.7
- **数据库**：PostgreSQL 13+（主数据库）+ Redis（缓存）
- **安全**：Spring Security
- **ORM**：Hibernate
- **构建工具**：Maven
- **JDK**：Java 17

### 2.2 系统架构
系统采用经典的分层架构设计：
- **表现层**：用户界面层，负责展示数据和接收用户操作
- **控制层**：处理HTTP请求，调用服务层完成业务逻辑
- **服务层**：封装核心业务逻辑，提供事务管理
- **数据访问层**：通过Spring Data JPA实现数据库操作
- **数据层**：PostgreSQL数据库存储系统数据

## 3. 功能需求

### 3.1 学生信息管理

#### 3.1.1 学生基本信息
- **功能**：学生信息的全面管理
- **具体功能**：
  - 学生信息录入、查询、修改
  - 学生信息导入/导出
  - 按学号、姓名、学院、专业等条件查询
  - 学生毕业状态管理
- **权限**：管理员可管理所有学生信息；学生可查看修改个人基本信息

#### 3.1.2 学业信息管理
- **功能**：管理学生成绩、学分、GPA
- **具体功能**：
  - 成绩录入、修改、查询
  - 成绩导入/导出
  - GPA自动计算
  - 学业历史记录查询
- **权限**：管理员可录入修改；学生可查询个人学业信息

### 3.2 奖学金评定管理

#### 3.2.1 奖学金类型管理
- **功能**：设置和管理奖学金基本信息
- **具体功能**：
  - 奖学金类型新增/修改/删除
  - 评定标准设置（金额、条件、申请时间等）
  - 奖学金状态管理（激活/停用）
- **权限**：管理员用户

#### 3.2.2 评定流程管理
- **功能**：实现完整的奖学金评审流程
- **具体功能**：
  - 资格初审和复审
  - 评审意见记录和管理
  - 评定结果生成和管理
- **权限**：管理员用户

### 3.3 公示管理
- **功能**：公示奖学金评定结果
- **具体功能**：
  - 公示内容生成和发布
  - 公示信息查询和浏览
  - 公示置顶和优先级管理
  - 公示信息软删除
- **权限**：管理员可发布、管理和删除公示；学生可查看公示

### 3.4 用户管理
- **功能**：系统用户的管理
- **具体功能**：
  - 用户账号新增/修改/删除
  - 用户角色分配
  - 用户登录日志记录
  - 用户密码重置
- **权限**：管理员用户

### 3.5 权限管理
- **功能**：基于角色的权限控制
- **具体功能**：
  - 角色定义和管理
  - 权限分配和控制
  - 菜单权限管理
- **权限**：管理员用户

### 3.6 系统管理
- **功能**：系统配置和维护
- **具体功能**：
  - 系统数据初始化
  - 系统参数配置
  - 操作日志管理
  - 数据备份和恢复
- **权限**：管理员用户

## 4. 用户角色与权限

### 4.1 学生用户
- 个人基本信息查看与修改
- 个人学业信息查询
- 奖学金公示结果查看
- 登录和退出系统

### 4.2 管理员用户
- 学生信息管理（录入、查询、修改、删除）
- 学业信息管理（成绩录入、修改、查询）
- 奖学金类型管理（新增、修改、删除）
- 奖学金评审流程管理
- 公示信息管理（发布、修改、删除）
- 用户管理（新增、修改、删除用户和角色）
- 系统配置和维护

### 4.3 教师用户（预留）
- 学生成绩录入和管理
- 奖学金评审参与

## 5. 数据实体
- **用户(User)**：用户ID、用户名、密码、角色、邮箱、电话等
- **学生(Student)**：学生ID、用户ID、学号、姓名、性别、学院、专业、班级等
- **奖学金类型(ScholarshipType)**：类型ID、名称、代码、金额、条件、有效期等
- **评审记录(Review)**：评审ID、学生ID、奖学金类型ID、评审状态、分数等
- **公示(Announcement)**：公示ID、标题、内容、发布时间、发布人、状态等
- **学业记录(AcademicRecord)**：记录ID、学生ID、学年、学期、课程信息、成绩等
- **权限(Permission)**：权限ID、权限名称、路径、描述等

## 6. 系统流程
1. 管理员初始化系统数据（用户、奖学金类型等）
2. 管理员设置奖学金类型和评定标准
3. 学生申请奖学金（预留功能）
4. 管理员进行资格初审和复审
5. 管理员记录评审意见和结果
6. 管理员发布奖学金评定结果公示
7. 学生查询公示结果
8. 管理员完成奖学金发放（预留功能）

## 7. 非功能需求

### 7.1 安全需求
- **用户认证**：基于用户名和密码的认证机制
- **权限控制**：基于角色的访问控制（RBAC）
- **数据加密**：敏感数据（如密码）加密存储
- **防止SQL注入**：使用参数化查询
- **防止XSS攻击**：输入验证和输出编码

### 7.2 性能需求
- 系统响应时间：一般操作<2秒，复杂查询<5秒
- 并发支持：支持100名用户同时在线操作
- 数据存储：支持至少10年的学生数据存储

### 7.3 可靠性需求
- 系统可用性：≥99.5%
- 数据备份：定期自动备份数据库
- 故障恢复：系统故障后能在1小时内恢复

### 7.4 可维护性需求
- 代码模块化：采用分层架构和模块化设计
- 日志记录：详细记录系统操作和错误信息
- 文档完整：提供完整的需求、设计和用户文档

## 8. 验收标准
### 8.1 功能验收
- 学生信息管理功能完整可用
- 奖学金评定流程完整实现
- 公示管理功能正常，包括软删除
- 用户和权限管理功能有效
- 系统数据初始化功能正常

### 8.2 性能验收
- 系统响应时间符合要求
- 并发用户数满足设计要求
- 数据查询和操作流畅

### 8.3 安全验收
- 用户认证和授权机制有效
- 敏感数据加密存储
- 系统能防御常见安全攻击
- 权限控制严格，无越权访问

## 9. 风险控制
- **数据安全风险**：加强数据加密和访问控制，定期备份数据
- **性能风险**：优化数据库查询，使用缓存技术提升性能
- **维护风险**：采用模块化设计，提高代码可维护性
- **安全风险**：定期进行安全审计，修复漏洞
- **兼容性风险**：使用成熟稳定的技术栈，确保系统兼容性
